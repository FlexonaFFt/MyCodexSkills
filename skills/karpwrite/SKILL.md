# Karpwrite

## Overview

Use this skill to keep writing work narrow, explicit, and purpose-driven. Prefer the smallest clear text that satisfies the user's goal, audience, constraints, and desired voice.

## Workflow

1. State assumptions before writing or editing.
2. Identify the text's goal, audience, format, tone, and constraints when they are known.
3. List plausible interpretations when the request is ambiguous.
4. Ask a clarifying question if choosing an interpretation would risk producing the wrong text.
5. Propose the simpler approach when one exists, with a short reason.
6. Define success criteria before drafting or revising.
7. Write or edit only what is needed.
8. Verify the result against the goal, audience, tone, length, and requested constraints.

## Writing Rules

- Write the minimum text that accomplishes the stated purpose.
- Avoid adding new claims, sections, examples, arguments, caveats, or stylistic flourishes unless the user asks for them or they are necessary.
- Prefer clear structure over decorative language.
- Preserve the user's intended meaning, stance, and voice when editing existing text.
- Match the existing style, terminology, formatting, and level of formality unless asked to change them.
- Remove only wording that became redundant, incorrect, or distracting because of your changes.
- If the draft becomes longer or more ornate than needed, shorten and clarify it before finishing.

## Editing Rules

- When editing existing text, change only what is required for the task.
- Do not "improve" unrelated wording, structure, punctuation, or formatting.
- Keep factual content intact unless the user asks for fact-checking or correction.
- Mark or explain material changes when the user needs to review intent, claims, or tone.
- For rewrites, preserve the core message before changing style.

## Goal-Driven Verification

- For new drafts, check that the text answers: who is it for, what should it do, and what should the reader understand or do next.
- For edits, check that the revised version still preserves the original intent unless the user requested a change in intent.
- For summaries, check that no central point is omitted and no unsupported claim is introduced.
- For persuasive or argumentative text, check that each claim supports the requested goal.
- For multi-step writing tasks, outline a short plan with a verification point for each step before executing.

## Communication Rules

- Be explicit about uncertainty.
- Do not silently choose between materially different audiences, tones, or formats.
- Keep explanations short unless the user asks for rationale.
- Stop and ask when the task is unclear enough that writing would be guesswork.
